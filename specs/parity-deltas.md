# DevinX Parity Deltas — intentional deviations from app.devin.ai

> Per spec §5.4.5: any place DevinX intentionally deviates from the web app
> gets one line here with the reason. If it's not logged, it's a bug.

| #   | Where                   | Deviation                                                                                                                                                       | Reason                                                                                                                                                                                                                 |
| --- | ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Session Board           | Blocked-first sorting (Needs input → Working → Recent → Sleeping)                                                                                               | Mobile triage is the product thesis (§7.4). Web app's sidebar "Recent" is chronological.                                                                                                                               |
| 2   | Empty states            | Terminal-prompt motif illustration + one CTA                                                                                                                    | Web app uses text-only empty states. Touch-first mobile benefits from a visual anchor (§5.3). Tone preserved.                                                                                                          |
| 3   | Long-press context menu | Pin / Archive / Terminate / Copy link / Add tag via iOS context menu                                                                                            | Web app has no equivalent (hover menus). Mobile-native mechanics, Devin skin (§5.4.4).                                                                                                                                 |
| 4   | Pull-to-refresh         | Native iOS pull-to-refresh on Board                                                                                                                             | Web app has no equivalent. Mobile-native mechanics (§5.4.4).                                                                                                                                                           |
| 5   | Haptics                 | Light impact on refresh, success on finished transition, warning on blocked                                                                                     | Web app has no haptics. Mobile-native mechanics (§5.4.4).                                                                                                                                                              |
| 6   | Composer "wake" warning | "Session is sleeping — sending will wake it." banner above composer                                                                                             | Web app auto-resumes silently. Mobile surfaces the behavior because it's the killer interaction (§7.3).                                                                                                                |
| 7   | Mono font               | `"SF Mono", "Roboto Mono", "JetBrains Mono", ...`                                                                                                               | Devin web uses `"SF Mono", "Roboto Mono", ui-monospace, monospace`. JetBrains Mono added as a bundled fallback for Android (SF Mono is Apple-only). First-listed font matches Devin on iOS.                            |
| 8   | Status label derivation | `deriveStatusLabel(session)` mirrors web app's state machine but runs client-side from `SessionResponse` enrichment fields                                      | Web app computes labels in React from the same fields. Same inputs → same labels. Not a deviation in output, only in execution location. Logged for transparency.                                                      |
| 9   | Home and Session Detail | Home uses a prominent centered Devin companion; Session Detail uses a compact companion that reflects canonical app/session state near the composer             | The web app has no equivalent persistent companion. Mobile gives the character a deliberate home-screen presence while keeping session work restrained, non-interactive, and subordinate to controls and content.      |
| 10  | Home and Session Board  | Home Recent follows its selected Cloud or paired-Mac destination; the full Session Board combines origins with Mac rows explicitly labeled and privacy-redacted | The Devin web app has no cross-device Desktop Bridge view. Destination-scoped Home context prevents one origin from displacing the other, while the archive retains a useful combined view and clear privacy boundary. |
